# CalmMD 设计备忘

## 定位

Local-first、reading-first、single-purpose 的 Markdown 阅读器。

## 功能准入标准

每个功能必须通过四个问题：
1. 是否明显提升长文阅读体验？
2. 是否保持 local-first？
3. 是否让内容更靠前，而不是工具更靠前？
4. 是否会把产品推向编辑器 / 知识库 / 协作平台？→ 如果会，不做。

## 交互约定

- 左侧：控制入口、最近打开、目录、进度
- 中间：正文主阅读区
- 内容 > 工具，留白 > 边框，节奏 > 装饰
- 主题切换柔和过渡，不闪断

## 不做

编辑、协作、云同步、用户系统、在线发布、知识图谱、插件系统。

## 可扩展

多文档阅读队列、注释/高亮、导出 PDF、脚注与提示块、Tauri 桌面封装。
